Veille Technologique

Sujet analysé : Infrastructure as Code

🎯 Problématique

Comment Terraform et Ansible permettent-ils, dans un environnement multi-cloud, de déployer une Infrastructure as Code à la fois efficace, sécurisée et maintenable ?

Cette question oriente ma veille technologique durant le BTS SIO SISR. Elle permet de comprendre les évolutions dans les infrastructures réseaux, leur automatisation, et les nouvelles compétences à maîtriser.

Illustration d’une architecture Infrastructure As Code

📘 Qu’est-ce que l'Infrastructure as Code ?

L’Infrastructure as Code consiste à décrire et gérer l’infrastructure sous forme de code versionné pour automatiser sa création, sa configuration et sa maintenance.

🔧 Objectifs du Infrastructure As Code

  • Automatiser les déploiements
  • Reproduire fidèlement les environnements
  • Garantir l’idempotence
  • Versionner et tracer les changements

🧱 Architecture Infrastructure As Code

  • Application Layer : outils de supervision
  • Control Layer : contrôleurs Infrastructure As Code (OpenDaylight, ONOS)
  • Infrastructure Layer : équipements réseau standards

👨‍💻 Impact pour le technicien SISR

  • Automatisation renforcée
  • Sécurité et gouvernance intégrées
  • Surveillance de l’état & gestion des dérives
  • Collaboration et documentation
